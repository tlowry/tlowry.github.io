Unlinking Disaster

Tony Lowry
2021-04-23
Linux Rhel

I recently noticed a script error on my part created a ton of dead links in /.
Deciding that because a script had created this mess I would use a script to fix it.

However my new creation had it’s own fatal flaw.
In a moment of stunning genius I had accidentally unlinked all directories in /

This resulted in no applications being runnable.
I couldn’t ssh into the machine or even open another terminal tab.

Just running ls produced this:

bash: /usr/bin/ls: /lib64/ld-linux-x86-64.so.2: bad ELF interpreter: No such file or directory

If this ever happens: do not close your terminal!
